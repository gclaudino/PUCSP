Cliente-servidor, TCP/IP e socket Uma arquitetura no estilo cliente-servidor com sockets TCP/IP contém os elementos destacados no diagrama de componentes (DCP) fig. 2.

A ideia de socket foi proposta em Berkeley, 1981. Ela abstrai a comunicação entre processos de uma rede no nível TCP/IP. Segue a conhecida filosofia “transmita sem ‘rezar’”—existe garantia de entrega dos pacotes transmitidos pela rede.

Fluxos de dados de um socket Um detalhe da comunicação por sockets na tecnologia Java é mostrado no diagrama da fig. 3.

Neste diagrama, observamos que objetos da classe Socket encontram-se associados a um fluxo de dados de entrada (InputStream) e a outro de saída (OutputStream). É por meio destes objetos que a comunicação entre dois sockets efetivamente acontece.

2 Experimento Cliente-Servidor (Sequencial)

Vamos projetar uma aplicação cliente-servidor capaz de informar a média em LP4 dos alunos, trabalhando em modo sequencial. A aplicação será constituída por dois componentes-aplicação.

2.1 Aplicação-Servidor

Nesta seção, implementaremos o componente-servidor.

2.1.2 Classe Servidor

A lógica efetiva de atendimento aos clientes encontra-se codificada nas classes Servidor e TarefaServidor. A primeira contém um ciclo perpétuo de atendimento a pedidos de conexão, marcada na instrução (2) da lst. 2. Ou seja, o fluxo de controle da aplicação-servidor entra em estado de pausa ao executar esta intrução. Quando chega um pedido de conexão, cria-se um objeto TarefaServidor para atendê-la (em modo sequencial).

Listagem 2: Classe servidor de médias LP4

// cs1::mf::Servidor::
import java.io.IOException;
import java.net.ServerSocket;
import java.net.Socket;

public class Servidor {

    public void run() throws IOException {
        ServerSocket ss = new ServerSocket(9001);
        int n = 0;
        while (true) {
            String nomeFluxo = Thread.currentThread().getName();
            String s = String.format("Aguardando conexões (%s)...", nomeFluxo);
            System.out.println(s);
            Socket cliente = ss.accept(); // <--(1) WAIT
            System.out.println("Atendendo cliente " + ++n); // <--(2)
            TarefaServidor t = new TarefaServidor(cliente);
            t.run();
        }
    }
}

Definição dos fluxos de dados do cliente Na construção da tarefa, associamos os dois fluxos de dados (de entrada e de saída) do socket de atendimento ao cliente. É por meio destes fluxos que ocorrerá a comunicação (lst. 4, lst. 5 e lst. 6).

Listagem 4: Associação com os fluxos de dados

    // cs1::mf::TarefaServidor::
    TarefaServidor(Socket c) throws IOException {
        fluxoEntrada = definirFluxoEntrada(c);
        fluxoSaida = definirFluxoSaida(c);
    }
    BufferedReader fluxoEntrada;
    PrintWriter fluxoSaida;

Listagem 5: Definição do fluxo de entrada

    // cs1::mf::TarefaServidor::
    BufferedReader definirFluxoEntrada(Socket cliente) throws IOException {
        InputStream is = cliente.getInputStream();
        InputStreamReader istream = new InputStreamReader(is);
        return new BufferedReader(istream);
    }

Listagem 6: Definição do fluxo de saída

    // cs1::mf::TarefaServidor::
    PrintWriter definirFluxoSaida(Socket cliente) throws IOException {
        OutputStream os = cliente.getOutputStream();
        BufferedOutputStream ostream = new BufferedOutputStream(os);
        // flush por write
        return new PrintWriter(os, true);
    }

Interpretador de comandos No método run() implementamos um interpretador de comandos enviados pelo cliente. No presente ensaio, apenas um comando será reconhecido: mf, interpretado como um pedido de acesso à media de LP4 (lst. 7).

Listagem 7: Lógica do intepretador de comandos do servidor

    // cs1::mf::TarefaServidor::
    public void run() throws IOException {
        String cmd = fluxoEntrada.readLine();
        if (cmd != null) {
            System.out.println(cmd);
            switch (cmd) {
                case "mf":
                    double mf = Math.random() * 10.0;
                    String resultado = String.format("%.1f", mf);
                    fluxoSaida.println(resultado);
                    break;
                default:
                    fluxoSaida.println("Não sei....");
            }
        }
    }

2.2.2 Classe Cliente

A lógica efetiva de solicitação de médias de LP4 encontra-se codificada nas classes Cliente e TarefaCliente. A primeira abre uma conexão contra o servidor e instancia um objeto que conhece a lógica de solicitação lst. 9.

Listagem 9: Classe cliente para solicitação de médias LP4

// cs1::mf::Cliente::
import java.io.IOException;
import java.net.Socket;

class Cliente {

    public void run() throws IOException {
        System.out.println("Conectando com o servidor de notas LP4...");
        try (Socket servidor = new Socket("localhost", 9001)) {
            TarefaCliente t = new TarefaCliente(servidor);
            t.run();
        } // servidor.close()
    }
}
