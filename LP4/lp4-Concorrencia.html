2 Contagem 1

2.1 Classe Tarefa

A tarefa a ser executada neste experimento (e nos demais desta série) é de geração da sequência ⟨1,2,⋯,10⟩\langle 1,2,\cdots,10 \rangle. As instruções que descrevem a lógica de geração serão executadas pelo fluxo JavaFX, o que deve ser evitado em situações de tarefas “demoradas”. Para simular uma tarefa deste tipo, acrescentaremos uma instrução que suspende a execução do objeto ativo contendo o fluxo de controle JavaFX (lst. 1).

Listagem 1: Lógica de geração da sequência de números

    // Tarefa (0.1):: 
    void contar() {
        // Lógica da tarefa a ser realizada
        String s = "--- Fluxo= " + Thread.currentThread().getName();
        System.out.println(s); // <--(1)
        for (int i = 1; i <= 10; i++) {
            String situacao = "Contando " + i + " de " + 10;
            situacaoLeg.setText(situacao); // <--(2)
            System.out.println(situacao);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                System.out.println("*** ERRO: " + e);
            }
        }
    }

A instrução (1) faz o objeto ativo identificar-se no terminal de execução Java. A instrução (2), foco primário deste experimento, força o objeto ativo a atualizar um elemento visual de interação com o usuário. Como, em seguida, iremos suspender este objeto por 1s, o usuário será impedido de interagir com a interface da aplicação.

Devido à instrução (2), precisaremos da interface Label (lst. 2).

Listagem 2: Interface exigida pela classe Tarefa

// Tarefa (0.1)::
import javafx.scene.control.Label;

e, também, associar a tarefa com este elemento visual em tempo de execução—e antes do início da realização da tarefa—conforme se executa o método associar() (lst. 3).

Listagem 3: Método para associar a tarefa ao fluxo JavaFX

    // Tarefa (0.1):: 
    void associar(Label s) {
        situacaoLeg = s;
    }
    Label situacaoLeg;

A execução deste método irá criar um novo fluxo de controle, responsável por executar as instruções da tarefa de contagem (lst. 17)

Listagem 17: Criação de um novo fluxo de controle, específico para contagem

    // AplContagem2 (0.2)::
    void contar(Tarefa c) {
        Runnable tarefa = () -> c.contar();
        Thread fluxoAdicional = new Thread(tarefa);
        // Encerro o fluxo de controle se a aplicação terminar
        fluxoAdicional.setDaemon(true);
        fluxoAdicional.start();
    }

É durante a realização da tarefa que deveremos observar um comportamento anômalo. Na tecnologia JavaFX, apenas o fluxo JavaFX pode executar instruções que se encontram nas classes de controle. Em particular, existe uma região neste experimento que deveria ser percorrida apenas por este fluxo: a região delimitada pelo objeto situacaoLeg da classe Label. Entretanto, programamos o fluxo de controle de contagem para também percorrer por aquela região de instruções. Isto será proibido pelo ambiente de execução JavaFX. Consequentemente, observaremos um erro em tempo de execução. No próximo experimento iremos resolver esta questão.

Listagem 41: Lógica de geração da sequência de números

    // Tarefa (0.5):: 
    void contar() {

        String s = "--- Fluxo= " + Thread.currentThread().getName();
        System.out.println(s);
        for (int i = 1; i <= 10; i++) {
            if (this.isCancelled()) { // <--(1)

            }
        }

Segundo o manual da tecnologia JavaFX:

In Java there is no reliable way to “kill” a thread in process. However, when cancel is called on a Task, it is important that the Task stop processing. A “run-away” Task might continue processing and updating the message, text, and progress properties even after the Task has been cancelled! In Java, cancelling a Task is a cooperative endeavor. The user of the Task will request that it be cancelled, and the author of the Task must check whether is has been cancelled within the body of the call method. There are two ways this can be done. First, the Task author may check the isCancelled method, inherited from FutureTask, to see whether the Task has been cancelled… https://docs.oracle.com/javase/8/javafx/api/javafx/concurrent/Task.html

6.3 Classe AplContagem5

Fluxo adicional O ponto central deste experimento encontra-se no método start() (lst. 46). Mais especificamente, na instrução destacada (1). Na tecnologia JavaFX, quando instanciamos um objeto Service, o ambiente de execução cria um novo fluxo de controle que irá correr pelas suas instruções. Desta maneira, interpretamos o objeto identificado por fluxoAdicional de duas maneiras. Por um lado, ele corresponde ao fluxo criado automaticamente pelo ambiente JavaFX. Por outro lado, ele contém a região de instruções que serão, por ele, executadas (métodos da classe Servico, neste caso).

Listagem 46: Ação inicial do fluxo JavaFX

    // AplContagem5 (0.5)::
    @Override
    public void start(Stage palco) {
        // Tarefa de contagem
        Servico fluxoAdicional = new Servico(); // <--(1)
        // Eventos do usuário
        Button contarEvt = definirBotaoContar(fluxoAdicional);
        Button pausarEvt = definirBotaoPausar(fluxoAdicional);
        Button continuarEvt = definirBotaoContinuar(fluxoAdicional);
        Button encerrarEvt = definirBotaoEncerrar(palco);
        Label situacaoLeg = definirTextoSituacao(fluxoAdicional);
        // Organzação dos elementos visuais
        HBox botaoSeq = new HBox(5, contarEvt, pausarEvt, continuarEvt, encerrarEvt);
        VBox raiz = new VBox(10, situacaoLeg, botaoSeq);
        // Cena principal
        Scene cena = new Scene(raiz);
        palco.setScene(cena);
        palco.setTitle("Contagem 5");
        palco.show();
    }

Listagem 52: Criação de um novo fluxo de controle, específico para contagem

    // AplContagem5 (0.5)::
    void contar(Servico fluxoAdicional) {
